# Quickstart Guide

Get Otto connected to your Linear and GitHub accounts in just a few steps.

## Prerequisites

Before you begin, make sure you have:

- A [Linear](https://linear.app) **or** [Jira](https://www.atlassian.com/software/jira) workspace with admin access
- A [GitHub](https://github.com) **or** [Azure DevOps](https://azure.microsoft.com/en-us/products/devops) organization with admin access
- An Otto account at [fullotto.app](https://fullotto.app)

## Step 1: Create an Organization

1. Log in to [fullotto.app](https://fullotto.app)
2. Click **Create Organization**
3. Enter your organization name
4. You'll be redirected to your new organization dashboard

![Create Organization](/screenshots/quickstart/01-create-organization.png)

## Step 2: Connect Linear

1. Go to **Settings → Integrations**
2. Click **Connect Linear**
3. Authorize Otto to access your Linear workspace
4. Select which teams Otto should have access to

![Connect Linear](/screenshots/quickstart/02-connect-linear.png)

## Step 3: Connect GitHub

1. In **Settings → Integrations**, click **Connect GitHub**
2. Install the Otto GitHub App on your organization
3. Select which repositories Otto should have access to

![Connect GitHub](/screenshots/quickstart/03-connect-github.png)

## Step 4: Create Your First Agent

1. Go to **Agents** in the sidebar
2. Click **Create Agent**
3. Configure your agent:
   - **Name**: e.g., "Backend Agent"
   - **Model**: Choose your preferred AI model
   - **System Prompt**: Add custom instructions for how your agent should behave
   - **Tags**: Organize agents with tags
4. Click **Save Changes**

![Create Agent](/screenshots/quickstart/04-create-agent.png)

## Step 5: Set Up a Trigger

1. Go to **Triggers** in the sidebar
2. Click **Create Trigger**
3. Configure:
   - **Source**: Linear or GitHub
   - **Event**: Choose the trigger event (e.g., "Issue Created")
   - **Agent**: Select the agent you created
   - **Project**: Specify which project(s) this trigger applies to
   - **Instructions**: Write custom instructions for when this trigger fires
4. Click **Create**

![Create Trigger](/screenshots/quickstart/05-create-trigger.png)

## You're Ready! 🎉

Now when you move a Linear ticket to "In Progress", Otto will:

1. Read the ticket requirements
2. Ask clarifying questions if needed
3. Create an implementation plan
4. Write the code
5. Open a pull request
