# PR Feedback Loop

Otto doesn't just create PRs — it responds to code review feedback and makes changes.

## How It Works

When someone leaves a review comment on an Otto-created PR:

1. **Otto reads the feedback** — Parses the comment and understands the request
2. **Makes changes** — Modifies the code as requested
3. **Pushes a new commit** — Updates the PR with the fixes
4. **Responds to the comment** — Explains what was changed

## Supported Feedback Types

### Code Changes

::code-group
```md [Review Comment]
Can you add input validation for the email field?
```

```md [Otto's Response]
Added email validation using class-validator. The endpoint now returns 400 if the email format is invalid.

Changes in this commit:
- Added `@IsEmail()` decorator to `CreateUserDto`
- Added error message: "Please provide a valid email address"
```
::

### Bug Fixes

::code-group
```md [Review Comment]
This will throw a null pointer exception if `user.profile` is undefined
```

```md [Otto's Response]
Good catch! Added optional chaining to handle the case where profile is undefined.

`user.profile?.avatar` → returns undefined instead of throwing
```
::

### Style/Convention Fixes

::code-group
```md [Review Comment]
Please use early returns instead of nested if statements
```

```md [Otto's Response]
Refactored to use early returns. The function is now easier to read.
```
::

## Best Practices

### Be Specific

Otto responds better to specific, actionable feedback:

| Feedback | Quality |
|----------|---------|
| "Add validation" | ❌ Vague |
| "Add email validation using class-validator" | ✅ Specific |
| "This looks wrong" | ❌ Vague |
| "This should return 404 instead of 500 when user not found" | ✅ Specific |

### One Topic Per Comment

Break complex feedback into separate comments for better results.

### Use Suggestions

GitHub's suggestion feature works great with Otto:

```suggestion
const user = await this.userService.findById(id);
if (!user) {
  throw new NotFoundException('User not found');
}
```

Otto will apply the suggestion directly.

## Limitations

Otto may struggle with:

- Architectural feedback requiring major refactoring
- Feedback referencing external context it doesn't have
- Subjective style preferences without clear direction

For complex changes, consider creating a new ticket instead.
